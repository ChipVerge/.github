RTL Design Engineer

RTL Design Full-time Remote 1–3 Years Experience
Apply Now
Role Overview

We are looking for an RTL Design Engineer to own design blocks end-to-end on production ASIC and FPGA targets. You will write synthesizable SystemVerilog, drive designs through lint, CDC, and synthesis flows, and be accountable for timing closure and quality of sign-off deliverables. This is a hands-on engineering role — you will work directly with the design and verification teams inside active project schedules. Disciplined coding standards and measurable outcomes are mandatory.

Responsibilities
  • Write and own synthesizable RTL in SystemVerilog for ASIC and FPGA targets
  • Implement digital blocks from micro-architecture specifications through synthesis handoff
  • Run and resolve lint, CDC, and formal equivalence checks independently
  • Define and review timing constraints; participate in timing closure iterations
  • Conduct and respond to RTL and micro-architecture peer reviews
  • Document design decisions, interface specifications, and block-level micro-architecture
Requirements
  • B.Sc. in Electrical Engineering, Electronics, or Computer Engineering (or equivalent)
  • 1–3 years of hands-on RTL design experience in an industrial or serious academic setting
  • Proficiency in SystemVerilog for synthesizable RTL — pipelines, FIFOs, arbiters, FSMs
  • Working knowledge of synthesis flows (Design Compiler, Genus, or Vivado)
  • Solid understanding of timing analysis: setup/hold, false paths, multi-cycle paths
  • Experience with CDC analysis and lint tools; ability to triage and resolve violations
  • Strong written communication in English for technical documentation and reviews
Good to Have
  • FPGA board bring-up experience (Xilinx, Intel/Altera, or similar)
  • Familiarity with formal verification tools (Cadence JasperGold, Synopsys VC Formal)
  • Exposure to low-power design techniques: clock gating, power domains, UPF
  • Awareness of DFT implications at the RTL stage (scan friendliness, isolation cells)
  • Python or Tcl scripting for design automation and regression
  • Any RTL project on GitHub demonstrating design judgment

DFT Engineer

DFT Full-time Remote 1–3 Years Experience
Apply Now
Role Overview

We are looking for a DFT Engineer to plan and implement testability structures on production ASIC designs. You will define DFT architecture, insert scan chains, run ATPG to coverage targets, implement memory BIST, and ensure DFT sign-off quality before tape-out. You need to understand both the structural requirements of test insertion and the verification impact — DFT-aware RTL decisions start at design time, not after synthesis.

Responsibilities
  • Define and implement DFT architecture: scan chains, compression, boundary scan, MBIST
  • Perform scan insertion and verify DRC-clean, lint-clean test-mode logic
  • Run ATPG (stuck-at, transition, path-delay) and achieve target fault coverage
  • Implement and verify memory BIST controllers and diagnose memory test failures
  • Review RTL for DFT friendliness; identify and resolve scan-blocking constructs early
  • Produce DFT reports, pattern sets, and test-coverage sign-off documentation
Requirements
  • B.Sc. in Electrical Engineering, Electronics, or Computer Engineering (or equivalent)
  • 1–3 years of hands-on DFT experience covering scan insertion and ATPG
  • Working knowledge of ATPG tools: Siemens Tessent, Synopsys DFT Compiler, or Cadence Modus
  • Understanding of scan architectures: full-scan, partial-scan, scan compression
  • Familiarity with MBIST architecture and memory repair flows
  • Ability to review RTL for test-mode isolation, clock-control, and tristate issues
  • Strong written communication for DFT plans and test coverage reports
Good to Have
  • Experience with IJTAG / IEEE 1687 instrument integration
  • Knowledge of boundary scan (JTAG / IEEE 1149.1) implementation and test
  • Familiarity with low-power DFT: power-domain-aware scan, always-on logic handling
  • Exposure to silicon debug and yield analysis using DFT infrastructure
  • Python or Tcl scripting for DFT flow automation and pattern management
  • Awareness of automotive-grade test requirements (ISO 26262 / AEC-Q100 context)

Custom Layout Engineer

Custom Layout Full-time Remote 1–3 Years Experience
Apply Now
Role Overview

We are looking for a Custom Layout Engineer to produce DRC/LVS-clean, performance-optimised full-custom layouts for analog and mixed-signal blocks. You will work from schematic to GDS in Cadence Virtuoso, applying rigorous matching, shielding, and parasitic-minimisation techniques. Blocks will include op-amps, ADCs, PLLs, bandgap references, and custom standard cells — each with silicon-proven quality expectations and no margin for layout-induced performance degradation.

Responsibilities
  • Perform full-custom layout of analog and mixed-signal blocks in Cadence Virtuoso

  • Ensure DRC and LVS clean for every deliverable — violations are not left for sign-off
  • Run parasitic extraction (QRC, StarRC, or Calibre xRC) and iterate for performance closure
  • Collaborate with analog designers to resolve layout-driven performance issues
  • Produce layout documentation covering floor-plan rationale and critical-net handling
Requirements
  • B.Sc. in Electrical Engineering or Electronics (or equivalent)
  • 1–3 years of full-custom analog/mixed-signal layout experience
  • Proficiency in Cadence Virtuoso Layout Suite (GXL or XL)
  • Solid understanding of analog layout fundamentals: matching, shielding, latch-up prevention
  • Hands-on experience with DRC/LVS sign-off using Calibre or Synopsys PVS
  • Experience with parasitic extraction and post-layout simulation correlation
  • Ability to read and interpret analog schematics and device-level circuit intent
Good to Have
  • Experience at advanced nodes (28nm, 16nm, or below) with complex DRC rule decks
  • SKILL or Python scripting for Virtuoso automation and parameterised cell generation
  • Familiarity with RF layout techniques: transmission lines, baluns, inductors
  • Knowledge of electromigration rules and metal stack selection for high-current paths
  • Exposure to library characterisation: timing, power, and noise (CCS/ECSM)
  • Any tapeout experience, even academic or shuttle projects, with demonstrated DRC/LVS closure
